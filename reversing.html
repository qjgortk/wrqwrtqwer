리버싱<Reverse-Engnieering>


리버싱이란?

  • 여러 분야에서 사용되는 용어로 인공물로부터
    설계 사상이나 지식을 추출하는 행위를 말하며 '역공학'이라고도 한다.
  • 리버싱은 통상적으로 컴파일된 바이너리(exe, dll, sys 등)를
    디스어셈블(disassemble) 도구를 이용하여 어셈블리 코드로 변환하여 필요한 작업을 수행하는 것이다.
※ 도움이 되는 사이트와 서적은 옆에 사진을 누르시면 됩니다.

리버싱에서 사용되는 기술

  1. 크랙

    • 프로그램의 제한된 기능을 제거시키는 것.

  2. 어셈블리

    • 소프트웨어 체인에서 가장 낮은 레벨
    • 리버싱의 언어

  3. 동적분석

    • 코드가 실행되지 않은 상태에서 보는 작업(감염은 되지 않음)

  4. 정적분석

    • 실제로 바이너리를 실행시켜서 감염시킨 뒤 상태를 보는 작업

자주 사용되는 공격기법

  1. 바이너리 분석

    • 소프트웨어의 바이너리를 직접 분석하여
      취약점을 찾거나 내부 로직을 이해하는 기법

  2. 디컴파일링

    • 컴파일된 실행파일이나 바이너리를 분석하여
      고급언어 수준으로 소스 코드와 유사한 형태로 복원하는 과정

  3. 코드 난독화 우회

    • 보안 강화를 위해 소스 코드를 난독화했더라도
      이를 역으로 해독하여 공격하는 기법
